Semester Project: Edge Computing Monitoring & Threat Detection

Project Overview

Our project aims to develop a distributed edge computing system that identifies people and detects threat situations (such as theft, fire, or vandalism) in real-time. The infrastructure combines Raspberry Pi single-board computers, AI camera modules, distributed backend clusters, containerized microservices, and notification bots.

Project Tasks

Task 1: Infrastructure In Progress

Deploying Ubuntu OS, object detection (YOLO), and setting up network PXE booting for worker nodes.

Task 2: HPL Benchmark To Do

Analyzing and benchmarking clustering performance using High Performance LINPACK.

Task 3: MPI Cluster To Do

Configuring Message Passing Interface (MPI) and exploring Amdahl's and Gustafson's Laws.

Task 4: Non-MPI Tools To Do

Verifying performance limits using a Task Distributor system.

Task 5: Monitoring To Do

Deploying Prometheus, Grafana, or CheckMK monitoring dashboards.

Task 6: AI Model To Do

Collecting images, annotating dataset, and training a custom TensorFlow/YOLO model.

Task 7: Backend Cluster To Do

Developing a containerized backend on k3s with distributed storage (Ceph/SeaweedFS/MinIO).

Task 8: Frontend To Do

Creating a React or Vue.js client displaying threat alerts, logs, and geolocation events.

Task 9: Telegram Bot To Do

Configuring real-time notification alerts for threat events and node health status.

Task 1: Infrastructure Setup

1. OS Installation

We chose Ubuntu Server 24.04 LTS for our Raspberry Pi 5 node. We flashed it using the official Raspberry Pi Imager tool.

2. Object Detection Software

To set up object detection, we installed YOLOv8 using Python. Here are the commands we ran on the Pi 5:

sudo apt update && sudo apt upgrade -y
pip3 install ultralytics

3. Camera Verification

To verify the Raspberry Pi AI Camera is working, we ran:
